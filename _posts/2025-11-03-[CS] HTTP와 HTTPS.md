# # HTTP 와 HTTPS

HTTP(HyperText Transfer Protocol)와 HTTPS(HyperText Transfer Protocol Secure)는 웹에서 데이터를 주고받기 위한 프로토콜이다. 이 둘의 주요 차이점은 보안에 있다.
<br /><br />

## 1. HTTP

HTTP는 데이터를 암호화하지 않고 전송하기 때문에, 중간에 누군가가 데이터를 가로채면 민감한 정보가 노출될 수 있다. 

<br />

### 1-1. HTTP의 특징

- **포트 번호**: 기본적으로 포트 80을 사용한다.

- **보안**: 데이터 암호화가 없기 때문에 보안에 취약하다.

- **속도**: 암호화 과정이 없기 때문에 HTTPS보다 빠르다.

<br /><br />

## 2. HTTPS

HTTPS는 SSL(Secure Sockets Layer) 또는 TLS(Transport Layer Security) 프로토콜을 사용하여 데이터를 암호화한다. 이를 통해 데이터가 전송되는 동안 중간에 가로채더라도 내용을 알 수 없게 만든다.

<br />

### 2-1. HTTPS의 특징

- **포트 번호**: 기본적으로 포트 443을 사용한다.

- **보안**: 데이터가 암호화되어 전송되므로 보안성이 높다.

- **속도**: 암호화 및 복호화 과정이 추가되기 때문에 HTTP보다 느릴 수 있다.

<br />

### 2-2. HTTPS 적용 방법

HTTPS를 사용하려면 웹 서버에 SSL/TLS 인증서를 설치해야 한다. 인증서는 신뢰할 수 있는 인증 기관(CA)에서 발급받을 수 있으며, 이를 통해 웹사이트의 신뢰성을 보장할 수 있다. CA에 인증서를 요청하면 CA 이름, 서버의 공개키, 서버의 정보를 활용하여 인증서를 생성하고 이를 CA 개인 키로 암호화하여 서버로 전송한다. 이때 인증서는 CA 개인 키로 암호화되니 신뢰성을 확보할 수 있다.

<br />

### 2-3. HTTPS의 동작 과정

1. 클라이언트가 서버에 HTTPS 요청을 보낸다.

2. 서버는 자신의 SSL/TLS 인증서를 클라이언트에 전송한다.

3. 클라이언트는 인증서를 검증하고, 세션 키를 생성하여 서버의 공개키로 암호화한 후 서버에 전송한다.

4. 서버는 자신의 개인키로 세션 키를 복호화한다.

5. 이후 클라이언트와 서버는 세션 키를 사용하여 데이터를 암호화하고 안전하게 통신한다.
