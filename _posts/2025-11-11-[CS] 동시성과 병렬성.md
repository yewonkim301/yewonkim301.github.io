# # 동시성과 병렬성


![img](/assets/img/til/cs/동시성과병렬성.png)

## # 동시성(Concurrency)

동시성은 여러 작업이 동시에 진행되는 것처럼 보이도록 하는 개념이다. 실제로는 하나의 프로세서가 빠르게 작업을 전환하면서 여러 작업을 처리하는 방식이다. 동시성은 주로 I/O 바운드 작업에서 유용하며, 시스템의 자원을 효율적으로 활용할 수 있게 한다.

<br />

### # 동시성의 특징

- **작업 전환**: 하나의 프로세서가 여러 작업을 빠르게 전환하면서 처리한다.

- **자원 효율성**: CPU가 I/O 작업을 기다리는 동안 다른 작업을 처리할 수 있어 자원을 효율적으로 사용할 수 있다.

- **복잡한 관리**: 동시성은 작업 간의 상태 관리와 동기화가 필요하여 복잡할 수 있다.

<br />

## # 병렬성(Parallelism)

병렬성은 여러 작업이 실제로 동시에 실행되는 개념이다. 이는 다중 프로세서나 다중 코어 시스템에서 가능하며, CPU 바운드 작업에서 특히 유용하다. 병렬성은 작업을 동시에 처리함으로써 전체 처리 시간을 단축할 수 있다.

<br />

### # 병렬성의 특징

- **동시 실행**: 여러 프로세서나 코어가 동시에 여러 작업을 처리한다.

- **성능 향상**: 작업을 병렬로 처리함으로써 전체 처리 시간을 줄일 수 있다.

- **복잡한 설계**: 병렬 처리는 작업 간의 데이터 공유와 동기화가 필요하여 설계가 복잡할 수 있다.

<br />

## # 동시성과 병렬성의 비교

| 특징         | 동시성 (Concurrency)                      | 병렬성 (Parallelism)                     |
|--------------|------------------------------------------|------------------------------------------|
| 정의         | 여러 작업이 동시에 진행되는 것처럼 보임   | 여러 작업이 실제로 동시에 실행됨                  |
| 실행 방식    | 하나의 프로세서가 작업을 빠르게 전환함     | 여러 프로세서나 코어가 동시에 작업을 처리함       |
| 주로 사용되는 작업 | I/O 바운드 작업                           | CPU 바운드 작업                           |
| 자원 활용    | 자원을 효율적으로 사용                     | 성능 향상을 위해 자원을 적극적으로 활용          |
| 복잡성       | 작업 간 상태 관리와 동기화 필요             | 데이터 공유와 동기화 필요                     |
